# Набор инструментов для датасета и детекции биллбордов

Этот проект помогает собрать датасет для детекции биллбордов и запускать обучение/инференс через удобный CLI.

В проект входят:
- `ooh_parser.py` для парсинга и авторазметки изображений наружной рекламы.
- `main.py` для интерактивного запуска обучения и детекции на отдельной фотографии.

## Для кого это

Если ты только начинаешь работать с YOLO и подготовкой данных, здесь есть полный практический цикл:
1. Собрать и авторазметить изображения.
2. Обучить модель на размеченных данных.
3. Запустить детекцию на своих фото и сохранить preview.

## Основные возможности

- Сбор изображений из нескольких источников (Bing, Yandex, DuckDuckGo, Wikimedia, Flickr, Openverse при доступности).
- Расширение запросов на русском и английском.
- Multi-object разметка (несколько биллбордов на одном изображении).
- YOLO-World как основной детектор, CV2 как fallback.
- Разделение данных на Accepted/Review/Trash для снижения количества мусорной разметки.
- Интерактивный лаунчер (`main.py`) для обучения и инференса.
- Повторное использование локальных `.pt` весов без постоянных перекачиваний.

## Структура проекта

- `ooh_parser.py`: сбор датасета и авторазметка.
- `main.py`: интерактивный запуск обучения/детекции.
- `dataset_ooh/accepted/images`: принятые изображения.
- `dataset_ooh/accepted/labels`: YOLO-лейблы.
- `dataset_ooh/accepted/dataset.yaml`: конфиг датасета для обучения.
- `dataset_ooh/accepted/preview`: preview-разметка.
- `dataset_ooh/review`: сомнительные образцы.
- `dataset_ooh/trash`: отклоненные изображения.

## Требования

- Python 3.10+
- macOS/Linux/Windows
- Для обучения желательно Apple Silicon MPS или CUDA GPU (на CPU тоже работает)

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Шаг 1: Сбор датасета через `ooh_parser.py`

Быстрый старт (интерактивный мастер):

```bash
python3 ooh_parser.py --wizard
```

Запуск через профиль:

```bash
python3 ooh_parser.py --count 1000 --profile balanced --detector auto --clean-existing
```

Полезные параметры:
- `--max-boxes-per-image` ограничивает число bbox на одном изображении.
- `--profile fast|balanced|quality` управляет балансом скорости и качества.
- `--clean-existing` пересканирует уже принятые изображения.
- `--only-clean` только пересканирует, без нового парсинга.

## Шаг 2: Обучение или инференс через `main.py`

Запуск:

```bash
python3 main.py
```

Дальше выбирается один из двух режимов:
1. Обучение на `dataset.yaml`.
2. Использование готовой YOLO `.pt` модели.

После выбора модели укажи путь к изображению.
Preview сохраняется рядом с исходной картинкой:
- `<имя_файла>_billboards_preview.jpg`

## Обучение с низкой нагрузкой (Apple M1, 8GB RAM)

В `main.py` добавлены облегченные настройки по умолчанию:
- профиль обучения `lite` (дефолт для train-режима)
- уменьшенный `imgsz`
- маленький `batch`
- сниженный `workers`
- уменьшенный `patience`
- `single_cls=True`, `cache=False`, `plots=False`, `amp=False`

Эти настройки снижают потребление памяти и общую нагрузку на систему.

## Использование уже скачанных моделей

И для обучения, и для инференса можно выбрать локальные `.pt` файлы из директории проекта.
Это позволяет не скачивать веса заново при каждом запуске.

В `main.py`:
- выбери локальную модель из списка,
- можно использовать уже скачанные веса, например `yolov8s-worldv2.pt` или `yolov8m-worldv2.pt`,
- можно указать путь к своей YOLO8/YOLO11/YOLO12 `.pt` модели.

## Решение проблем

Если модель YOLO не загружается:
- проверь активирован ли venv: `source .venv/bin/activate`
- переустанови зависимости: `pip install -r requirements.txt`
- выбери другой локальный `.pt` файл в `main.py`

Если URL собирается слишком мало:
- увеличь `--pool-multiplier`
- расширь запросы через `--queries`
- перезапусти скрипт (он продолжает с текущего прогресса)

Если в разметке всё еще есть шум:
- вручную дочисти `review/`
- переобучи модель после чистки
